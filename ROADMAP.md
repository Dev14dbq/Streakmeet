# StreakMeet - Подробный Roadmap & Architecture

## Технический стек

- **Frontend:** React 18+, TypeScript, Vite, Tailwind CSS, Zustand (стейт-менеджмент), React Query.
- **Mobile:** Capacitor (доступ к Native Camera, Push, File System), PWA Manifest.
- **Backend (Core API):** Node.js (NestJS или Express), TypeScript, Prisma ORM.
- **Backend (ML/CV Worker):** Python (FastAPI), OpenCV, InsightFace/ArcFace (распознавание), Anti-Spoofing модели.
- **База данных & Кэш:** PostgreSQL (основная БД), Redis (очереди задач BullMQ/Celery, кэш сессий).
- **Инфраструктура:** Docker, S3-совместимое хранилище (AWS S3 / Cloudflare R2 / MinIO), Firebase (Push & Ads).

---

## Детализированный план разработки (Checklist)

### Фаза 1: Инфраструктура и Базовый Setup

- [ ] Инициализировать монорепозиторий (Turborepo или npm workspaces).
- [ ] Настроить ESLint, Prettier, Husky (pre-commit хуки).
- [ ] **Frontend:** Развернуть React + Vite, настроить Tailwind CSS и роутинг (React Router).
- [ ] **Frontend:** Подключить Capacitor, инициализировать iOS и Android проекты.
- [ ] **Backend:** Развернуть Node.js сервер, подключить Prisma, описать базовую схему БД.
- [ ] **Backend:** Поднять PostgreSQL и Redis через `docker-compose`.
- [ ] **Storage:** Настроить S3 клиент для загрузки медиафайлов.

### Фаза 2: База Данных и Авторизация

- [ ] Спроектировать таблицы: `users`, `friendships`, `streaks`, `streak_days`, `meet_proofs`, `gem_transactions`.
- [ ] Реализовать регистрацию/логин (JWT токены + Refresh токены).
- [ ] Создать API профиля пользователя (установка никнейма, аватара, часового пояса).
- [ ] Настроить генерацию уникального ID для QR-кода при регистрации.

### Фаза 3: Социальный Граф (Друзья)

- [ ] **UI:** Экран профиля с личным QR-кодом (библиотека `react-qr-code`).
- [ ] **UI:** Сканер QR-кодов (через `@capacitor-community/barcode-scanner`).
- [ ] **API:** Поиск пользователей по точному совпадению никнейма.
- [ ] **API:** Отправка запроса в друзья, принятие, отклонение, удаление из друзей.
- [ ] **UI:** Вкладка "Друзья" с разделением на "Мои друзья" и "Входящие запросы".

### Фаза 4: ML Worker и Регистрация Лица (Enrollment)

- [ ] **ML Worker:** Поднять Python FastAPI сервис с моделью детекции лиц (RetinaFace) и извлечения эмбеддингов (ArcFace).
- [ ] **ML Worker:** Интегрировать алгоритм Liveness (проверка на фото/экран).
- [ ] **API:** Эндпоинт генерации Challenge для Liveness (например, "поверни голову вправо").
- [ ] **UI:** Экран регистрации лица (камера с оверлеем-маской).
- [ ] **API:** Сохранение эталонного вектора лица (embedding) в БД (зашифровано).

### Фаза 5: Ядро Серий (Streaks)

- [ ] **API:** Создание серии с другом (проверка, что оба согласны).
- [ ] **Логика:** Система учета часовых поясов (определение полуночи для конкретной пары).
- [ ] **API:** Загрузка совместного фото (Meet Proof).
- [ ] **ML Worker:** Проверка фото (наличие 2+ лиц, сравнение эмбеддингов с эталонами участников, Liveness).
- [ ] **API:** Запись успешного дня (`streak_days`), обновление счетчика серии.
- [ ] **Cron:** Ежедневная джоба (каждый час проверяет часовые пояса), которая сбрасывает серию, если не было фото.

### Фаза 6: Экономика Гемов (Ledger)

- [ ] **API:** Создать строгую систему транзакций (Ledger) для гемов, чтобы избежать багов с балансом.
- [ ] **Логика:** Начисление +1 гема каждому при успешном Meet Proof.
- [ ] **Логика:** Начисление бонусных гемов за юбилеи (7, 30, 50, 100 дней).
- [ ] **API:** Эндпоинт "Заморозить серию" (-5 гемов, статус дня меняется на `frozen`).
- [ ] **API:** Эндпоинт "Спасти друга" (списание 5 гемов у инициатора, заморозка серии для двоих).
- [ ] **Монетизация:** Интеграция AdMob (Rewarded Video) -> просмотр рекламы вызывает webhook -> начисление +1 гема.

### Фаза 7: UI/UX, Камера и Геймификация

- [ ] **UI:** Главный экран со списком активных серий (сортировка по времени до сгорания).
- [ ] **UI:** Экран конкретной серии (календарь встреч, галерея совместных фото).
- [ ] **UI:** Кастомный интерфейс камеры для совместного фото (переключение фронт/тыл, вспышка).
- [ ] **Геймификация:** Визуальные статусы огня (SVG/Lottie анимации в зависимости от длины серии: серая, костер, неон, черное пламя).
- [ ] **Медиа:** Наложение водяного знака (Day X + логотип) на фото перед сохранением.
- [ ] **UI:** Кнопка "Поделиться" (вызов Capacitor Share API для отправки в Telegram/Insta).

### Фаза 8: Уведомления и Удержание (Retention)

- [ ] Настроить Firebase Cloud Messaging (FCM) и APNs сертификаты.
- [ ] **Cron/Push:** Утреннее напоминание ("У вас 14 часов...").
- [ ] **Cron/Push:** Вечернее предупреждение ("Серия сгорает через 4 часа!").
- [ ] **Cron/Push:** КРИТИЧЕСКОЕ предупреждение за 1 час до полуночи.
- [ ] **Push:** Уведомление "Друг спас вашу серию!".
- [ ] **Медиа:** Фоновый воркер (FFmpeg) для склейки 30 фото в Timelapse-видео ("Капсула времени").

### Фаза 9: Безопасность, Тестирование и Релиз

- [ ] **Security:** Rate limiting на API, защита от брутфорса OTP/Паролей.
- [ ] **Legal:** Составление Terms of Service и Privacy Policy (GDPR/Биометрия).
- [ ] **UI:** Окно явного согласия на обработку биометрических данных при регистрации.
- [ ] **PWA:** Настройка `manifest.json` и Service Worker для установки из браузера.
- [ ] **iOS:** Настройка Xcode, добавление `NSCameraUsageDescription`, `NSFaceIDUsageDescription`.
- [ ] **Android:** Настройка `AndroidManifest.xml` (разрешения на камеру, пуши).
- [ ] Подготовка скриншотов и публикация в App Store и Google Play.
